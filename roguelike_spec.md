## 1. ゲーム概要

| 項目 | 内容 |
|---|---|
| ジャンル | 2Dグリッド型ローグライク |
| ターン制 | 完全ターン制（プレイヤー行動 → 敵全体行動 で1ターン） |
| プレイ人数 | 1人 |
| プラットフォーム | **Python / ターミナル**（CLI）|
| 実装言語 | Python 3.x【要確定：最低バージョン】 |

### 1.1 ゲームの目的
プレイヤーは **30ターン生き残ること** でゲームをクリアする。HPが0以下になると敗北となる。

敵の全滅はスコア加算対象（§12参照）だが、**勝利条件ではない**。30ターン経過前に全敵を撃破してもゲームは継続し、規定ターン数の経過を待つ必要がある。

### 1.2 実行環境
| 項目 | 仕様 |
|---|---|
| 実行形式 | コマンドラインから `python main.py` 等で起動 |
| 表示方式 | ターミナル（標準出力）に等幅フォントで描画 |
| 入力方式 | キーボード（標準入力） |
| OS | クロスプラットフォーム想定（Windows / macOS / Linux）【要確定】 |

> 詳細な表示仕様・入力仕様は §15（実行環境・表示仕様）参照。

## 2. 用語定義

| 用語 | 定義 |
|---|---|
| マップ | 10×10のグリッド空間 |
| マス | マップを構成する最小単位の正方形領域。座標 (x, y) で一意に識別される。 |
| ターン | プレイヤーが1回行動し、続いて全敵が1回行動する一連の流れ |
| エンティティ | プレイヤーまたは敵の総称 |
| 戦闘 | プレイヤーと敵が同一マスに存在する状態で発生するダメージ交換処理 |

## 3. 勝敗条件

### 3.1 勝利条件
**生存型**：規定ターン数 `SURVIVE_TURNS` を経過した時点でプレイヤーが生存している（HP > 0）こと。

`SURVIVE_TURNS` はエリアサイズによって異なる（§4.6 参照）。

| エリア | `SURVIVE_TURNS` |
|---|---|
| small | 30 |
| medium | 100 |
| large | 200 |

判定タイミング：各ターンの「ターンカウント +1」処理後（§7.1 [7] のタイミング）。
- 経過ターン数 ≥ `SURVIVE_TURNS` かつ プレイヤーHP > 0 → クリア
- それ以前に HP ≤ 0 → 敗北（§3.2）

### 3.2 敗北条件
プレイヤーHPが0以下になる。

### 3.3 同時成立時の優先順位
同一ターン内で勝利条件と敗北条件が同時に成立した場合は **敗北を優先**する（HP判定が先）。

## 4. 座標系・マップ仕様

### 4.1 座標系
- 原点 `(0, 0)` は左上
- x軸：右方向に増加
- y軸：下方向に増加
- 有効範囲：`0 ≤ x ≤ 9` かつ `0 ≤ y ≤ 9`

### 4.2 マップサイズ
マップサイズはエリアサイズ設定（§4.6）によって決まる。

| エリア | 幅 (MAP_W) | 高さ (MAP_H) | 総マス数 |
|---|---|---|---|
| small | 8 | 8 | 64 |
| medium | 10 | 10 | 100 |
| large | 15 | 15 | 225 |

### 4.3 壁
マップ内に通行不可の壁マスを配置する。壁数の範囲はエリアサイズ（§4.6）によって異なる。

| 項目 | 値 |
|---|---|
| 配置方式 | ランダム（ゲーム開始時に毎回生成） |
| 壁マス数（通常生成） | 総マス数の **15〜22%** を目安（エリアごとの絶対値は §4.6 参照） |
| 壁マス数（フォールバック生成） | 例外的に少ない値を許容（§4.4.2 参照） |
| 壁マスの状態 | 通行不可。プレイヤー・敵ともに進入できない |

> ※ 壁マス数が通常範囲外となるのは、§4.4.2 のフォールバック処理が発動した場合のみ。

### 4.4 マップ生成の整合性ルール
壁配置の結果、ゲーム進行が不可能になることを防ぐため、以下のすべてのルールを満たす必要がある。違反した場合は **マップ全体を再生成** する。

| ID | ルール | 検証方法 | 通常生成 | フォールバック生成 |
|---|---|---|---|---|
| MAP-00 | 壁マス数が指定範囲内 | 壁マス数のカウント | 15 ≤ 壁数 ≤ 20 | 0 / 5 / 10 のいずれか（§4.4.2） |
| MAP-01 | プレイヤー初期位置は壁マスでない | 座標確認 | 必須 | 必須 |
| MAP-02 | 全敵の初期位置は壁マスでない | 座標確認 | 必須 | 必須 |
| MAP-03 | プレイヤー初期位置から **すべての敵** へ経路として到達可能 | プレイヤー位置から幅優先探索（BFS）で到達可能マスを列挙し、全敵がその集合に含まれるか確認 | 必須 | 必須 |
| MAP-04 | プレイヤー初期位置の周囲4マスがすべて壁または範囲外でない（詰み配置回避） | プレイヤー初期位置の上下左右のうち、最低1マスが移動可能であること | 必須 | 必須 |
| MAP-05 | 全敵の初期位置の周囲4マスについても同様（任意の敵が初手から動けることを保証） | 各敵について上下左右の最低1マスが移動可能であること | 必須 | 必須 |

> ※ MAP-00 のみフォールバック生成時に基準が緩和される。MAP-01〜MAP-05 はゲーム進行の前提条件であるため、通常生成・フォールバック生成のいずれでも違反は許容されない。

#### 4.4.1 再生成のリトライ仕様
| 項目 | 値 | 備考 |
|---|---|---|
| 最大リトライ回数 | `MAP_GEN_MAX_RETRY = 100` | 実装後調整可。100回以内でほぼ確実に成立する想定 |
| 上限到達時の挙動 | **フォールバック処理** | 下記 §4.4.2 参照 |

#### 4.4.2 フォールバック処理
MAP_GEN_MAX_RETRY 回連続でMAP-01〜05のいずれかに違反し続けた場合、以下の手順でマップを生成する。エラー終了はしない。

1. 壁マス数を **下限値（15マス）** に固定して再生成を試みる（最大 `MAP_GEN_MAX_RETRY` 回）
2. それでも失敗する場合、壁マス数をさらに減らす（5マス刻みで段階的に減らし、最低0マスまで）
3. 壁0マス（壁なし状態）であればMAP-01〜05は確実に満たせるため、必ずマップが成立する

| 段階 | 壁マス数 | リトライ回数 |
|---|---|---|
| 通常 | 15〜20 | 最大100回 |
| フォールバック1 | 15固定 | 最大100回 |
| フォールバック2 | 10 | 最大100回 |
| フォールバック3 | 5 | 最大100回 |
| 最終フォールバック | 0 | 1回（必ず成功） |

> ※ 通常運用ではフォールバック処理に到達することは想定されていないが、安全装置として実装する。フォールバック発動はログに記録し、QAで頻発する場合はパラメータ調整の対象とする。

### 4.5 初期配置ルール
- プレイヤー・敵は壁マスを除いた座標からランダムに選ばれる
- すべてのエンティティは互いに重ならない座標に配置される
- 初期配置時にプレイヤーと敵が同一マスにいることはない

### 4.6 エリアサイズ設定

`config.py` の `CURRENT_AREA` を変更することで、マップ規模と関連パラメータを一括切り替えできる。

| パラメータ | small | medium | large |
|---|---|---|---|
| MAP_W × MAP_H | 8 × 8（64マス） | 10 × 10（100マス） | 15 × 15（225マス） |
| WALL_MIN | 10 | 15 | 34 |
| WALL_MAX | 14 | 22 | 50 |
| 壁割合（目安） | 15.6〜21.9% | 15.0〜22.0% | 15.1〜22.2% |
| SURVIVE_TURNS | 30 | 100 | 200 |
| MAX_ACTIVE_ENEMIES | 2 | 3 | 5 |
| ENEMY_SPAWN_INTERVAL | 5 | 5 | 5 |
| INITIAL_ITEM_COUNT | 3 | 5 | 8 |

> ※ `CURRENT_PRESET`（easy / normal / hard）を同時に指定すると SURVIVE_TURNS をさらに上書きできる。その他のパラメータ（PLAYER_MAX_HP, ENEMY_ATK 等）はエリアサイズに依存せず `_DEFAULTS` の値が使われる。
>
> ※ `INITIAL_ITEM_COUNT` はすべてのエリアで `MAX_INVENTORY（=2）+ 1` 以上を確保している。これにより「インベントリ満タンで拾えない」シナリオ（§11.3 上限到達時の挙動）を必ず検証できる。

## 6. 操作仕様

### 6.1 入力キー
| キー | 機能 | 効果 | ターン消費 |
|---|---|---|---|
| `W` | 上に移動 | `y -= 1` | あり（成立時） |
| `S` | 下に移動 | `y += 1` | あり（成立時） |
| `A` | 左に移動 | `x -= 1` | あり（成立時） |
| `D` | 右に移動 | `x += 1` | あり（成立時） |
| `H` | 回復薬使用 | HP回復（§11.4） | あり（成立時） |
| `X` | 回復薬を1つ破棄 | インベントリ -1（HP変動なし） | あり（成立時） |
| `R` | リスタート | 初期化状態へ遷移 | （ゲーム終了状態のみ有効） |

### 6.2 入力受付
- 移動・回復薬使用・破棄キー：**プレイ中** かつ プレイヤーターン中のみ受付
- リスタートキー：**クリア / ゲームオーバー** 状態のみ受付
- 上記以外の状態でのキー入力、または定義外キー入力は **無視**（ターン消費なし、状態変化なし）

### 6.3 移動の正常系
- 移動先がマップ範囲内かつ壁でない → 移動成立、ターン消費
- 移動先に敵がいる（壁でない） → 移動成立、戦闘発生（§8）、ターン消費

### 6.4 移動の異常系
| ケース | 処理 |
|---|---|
| 移動先がマップ範囲外（`x < 0` / `x > 9` / `y < 0` / `y > 9`） | 移動不成立。ターン消費しない、再入力受付 |
| 移動先が壁マス | 移動不成立。ターン消費しない、再入力受付 |
| 未定義キー入力 | 無視。ターン消費しない |
| プレイヤー敗北後の入力 | 無視 |

## 7. ターン進行仕様

### 7.1 1ターンの流れ
```
[1] プレイヤー入力受付
 ↓
[2] プレイヤー行動処理
     (a) 移動 / 戦闘 / 回復薬使用 / 回復薬破棄 のいずれか
     (b) 移動成立時、移動先マスに回復薬があり、かつ インベントリ < MAX_INVENTORY の場合に自動取得
         （インベントリ +1、マスからアイテム除去、HPは変動しない）
     (c) インベントリが満タンの場合、マス上のアイテムは残存し、自動取得は発生しない
 ↓
[3] 勝敗判定（敗北チェック → 勝利チェック）
 ↓ (継続の場合)
[4] 敵全体の行動処理（ランダム移動 / 戦闘）
 ↓
[5] 勝敗判定（敗北チェック → 勝利チェック）
 ↓
[6] ターンカウント +1
 ↓
[6b] 敵スポーン判定（§16）
     turn % ENEMY_SPAWN_INTERVAL == 0 かつ 生存敵数 < MAX_ACTIVE_ENEMIES の場合に敵を1体追加
 ↓
[7] 勝敗判定（敗北チェック → 勝利チェック）
 ↓ ターン[1]へ
```

> ※ 回復薬使用（H）・破棄（X）は移動・戦闘とは独立した行動で、それぞれ1ターンを消費する（[2]の選択肢）。
>
> ※ 勝敗判定は計3回行われる（[3] [5] [7]）。各タイミングで敗北チェックを先行させ、その後に勝利チェックを行う（§3.3 同時成立時の優先順位）。
>
> ※ [6b] の敵スポーンは勝敗判定 [7] より前に実行されるため、スポーンされた敵は次ターンの [4] から行動を開始する（スポーンしたターンは行動しない）。
>
> ※ [7] のタイミングが生存型の勝利判定（ターン数 ≥ SURVIVE_TURNS）の主要判定ポイントとなる。

### 7.2 敵の行動ルール

#### 7.2.1 行動回数
- 各敵は1ターンに1回行動する。

#### 7.2.2 移動先抽選アルゴリズム（ゆるい追尾）
各ターン、以下の2段階で移動先を決定する（§17 ENEMY_CHASE_RATE 参照）。

**段階 1（追尾モード、確率 ENEMY_CHASE_RATE）**
1. `random.random() < ENEMY_CHASE_RATE` の場合、追尾モードに入る。
2. 上下左右の4方向のうち、移動後のマンハッタン距離が現在より小さくなる方向を追尾候補とする。
3. 追尾候補から「無効マス」を除外する（無効マスの定義は下記）。プレイヤーのマスは有効。
4. 有効な追尾候補が1つ以上あれば、その中からランダムに1方向を選択して移動する。
5. 有効な追尾候補がない場合 → 段階 2（ランダム移動）にフォールバックする。

**段階 2（ランダム移動、段階 1 に該当しない場合 / フォールバック）**
1. 上下左右の4方向をランダムにシャッフルして順に試行する。
2. 移動先が「無効マス」でなければそこへ移動する。
3. 4方向すべてが無効マスの場合 → その敵はそのターン移動しない（静止）。

**無効マス**
| 条件 | 理由 |
|---|---|
| マップ範囲外（`x < 0` / `x ≥ MAP_W` 等） | 範囲外 |
| 壁マス（`#`） | 通行不可 |
| 他の敵が存在するマス | 重なり不許可（§7.2.4） |

> プレイヤーのマスは無効マスに含まれない（§7.2.3）。

| 定数 | 値 | 備考 |
|---|---|---|
| `ENEMY_CHASE_RATE` | 0.7 | 追尾モードに入る確率（§17 で設定） |

#### 7.2.3 戦闘の発生
- 追尾モード・ランダム移動いずれの場合も、移動先にプレイヤーがいる場合 → **有効な移動先として確定**し、戦闘発生（§8）
- プレイヤーマスは「無効マス」に含まれない（敵はプレイヤーに突撃できる）

#### 7.2.4 敵同士の重なり
- 敵同士の同一マス占有は **不許可**
- 既に他の敵がいるマスは無効マスとして扱う（§7.2.2 の再抽選対象）
- 戦闘前後でも敵同士は重ならない（撃破直後に空いたマスへ別の敵が移動するのはOK）

## 8. 戦闘仕様

### 8.1 発生条件
プレイヤーと敵が同一マス `(x, y)` に存在する瞬間に発生。

### 8.2 ダメージ計算
ランダム要素として **ミス（回避）** と **クリティカル** を導入する。攻撃時、以下の順で判定する。

#### 8.2.1 判定パラメータ
| 定数 | 値 | 備考 |
|---|---|---|
| `MISS_RATE` | 10% | ミス発生確率（暫定、実装後調整可） |
| `CRITICAL_RATE` | 20% | クリティカル発生確率（暫定、実装後調整可） |
| `CRITICAL_MULTIPLIER` | 2.0 | クリティカル時のダメージ倍率 |
| `DAMAGE_VARIANCE_RATE` | 0.2 | 通常ダメージの振れ幅（ATK の ± この割合; 0.0=固定値） |

#### 8.2.2 判定順序とダメージ
1回の攻撃につき以下を順に判定する：

```
[1] ミス判定（確率 MISS_RATE = 10%）
    - 成立 → ダメージ = 0、判定終了
    - 不成立 → [2]へ
[2] 基礎ダメージ決定
    - base = max(1, round(ATK × uniform(1 - VARIANCE_RATE, 1 + VARIANCE_RATE)))
    - ATK=20, VARIANCE=0.2 → base ∈ [16, 24]（整数）
    - ATK=10, VARIANCE=0.2 → base ∈ [8, 12]（整数）
    - VARIANCE=0.0 → base = ATK（従来通り固定値）
[3] クリティカル判定（確率 CRITICAL_RATE = 20%）
    - 成立 → ダメージ = int(base × CRITICAL_MULTIPLIER)
    - 不成立 → ダメージ = base（通常攻撃）
```

#### 8.2.3 適用範囲
ミス・クリティカル・ダメージ振れ幅ともに **プレイヤー → 敵、敵 → プレイヤーの双方向に適用** する。

| 攻撃方向 | ミス適用 | クリティカル適用 | ダメージ振れ幅適用 |
|---|---|---|---|
| プレイヤー → 敵 | ○ | ○ | ○ |
| 敵 → プレイヤー | ○ | ○ | ○ |

#### 8.2.4 ダメージパターン一覧（DAMAGE_VARIANCE_RATE=0.2 時）
| 結果 | 確率 | プレイヤー攻撃ダメージ（ATK=20） | 敵攻撃ダメージ（ATK=10） |
|---|---|---|---|
| ミス | 10% | 0 | 0 |
| 通常 | 72%（= 90% × 80%） | 16〜24 | 8〜12 |
| クリティカル | 18%（= 90% × 20%） | 32〜48（= 通常×2） | 16〜24（= 通常×2） |

> ※ 各確率は独立判定の結果であり、合計100%となる。
> ※ `DAMAGE_VARIANCE_RATE = 0.0` の場合、通常ダメージは ATK 固定値（v1.x 以前の挙動）になる。

### 8.3 戦闘処理順序
1. 攻撃側（移動してきた側）の攻撃を先に解決（ミス/クリ判定 → ダメージ適用）
2. 被攻撃側のHPが0以下になった場合 → 撃破処理し、反撃は発生しない
3. 被攻撃側が生存していれば反撃（同様にミス/クリ判定 → ダメージ適用）

| 戦闘発生パターン | 攻撃順 |
|---|---|
| プレイヤーが敵のマスに進入 | プレイヤー攻撃 → (敵生存なら) 敵反撃 |
| 敵がプレイヤーのマスに進入 | 敵攻撃 → (プレイヤー生存なら) プレイヤー反撃 |

### 8.4 撃破処理
- 敵HP ≤ 0 → 当該敵をマップから除去
- プレイヤーHP ≤ 0 → ゲームオーバー（§3.2）

#### 8.4.1 HP表示・内部値のクランプ
| 項目 | 仕様 |
|---|---|
| 内部HP値 | 計算結果がマイナスでもそのまま保持しない。0 でクランプする |
| 表示HP値 | 0 未満は 0 として表示 |
| 上限 | MAX_HP を超える回復は MAX_HP でクランプ（§11.4 / §11.8 参照） |

> 例：HP = 5 のときに 20 ダメージ受けた場合、内部HPは -15 ではなく **0** として扱われる。敗北判定（HP ≤ 0）は変わらず成立する。

### 8.5 戦闘の境界値
| 観点 | 境界 | 期待結果 |
|---|---|---|
| 敵HPちょうど0 | HP = 0 | 撃破成立、反撃なし |
| 敵HPあと1で撃破 | HP = 1, 受ダメ ≥ 1 | 撃破成立 |
| ミスでダメージ0 | 攻撃側がミス判定成立 | HP変動なし、相手は反撃可能 |
| クリティカルで一撃撃破（最小クリ） | 敵HP=30, プレイヤー攻撃クリ最小=32（base=16×2） | 撃破成立（32 > 30） |
| クリティカル最大ダメージ | プレイヤーATK=20, base=24, クリ倍率2 | 48ダメージ |
| 通常攻撃最小ダメージ | ATK=20, base最小 | 16（DAMAGE_VARIANCE_RATE=0.2 時） |
| 通常攻撃最大ダメージ | ATK=20, base最大 | 24（DAMAGE_VARIANCE_RATE=0.2 時） |
| VARIANCE=0 の固定ダメージ | DAMAGE_VARIANCE_RATE=0.0 | 通常ダメージ常に ATK（20/10） |
| プレイヤーHPちょうど0 | HP = 0 | 敗北成立 |
| プレイヤーHPがマイナス計算 | HP=5 で20ダメージ受ける | 敗北成立、HP内部値・表示ともに **0** にクランプ |
| 連続ミスでターン経過 | 双方ミスが連続 | HPは双方変動せず、ターン進行 |

## 9. 状態遷移

### 9.1 ゲーム全体の状態
```
[初期化] → [プレイ中] → [クリア] ─┐
              │                  │
              └──────→ [ゲームオーバー] ─┤
                                          │
                  ┌───────────────────────┘
                  │ リスタート(R)
                  ↓
              [初期化]
```

| 状態 | 遷移トリガ | 遷移先 |
|---|---|---|
| 初期化 | マップ生成・配置完了 | プレイ中 |
| プレイ中 | プレイヤーHP ≤ 0 | ゲームオーバー |
| プレイ中 | ターン数 ≥ 30 かつ プレイヤーHP > 0 | クリア |
| クリア | `R` キー入力 | 初期化（マップ・敵・スコア・インベントリ全リセット） |
| ゲームオーバー | `R` キー入力 | 初期化（同上） |

#### 9.1.1 リスタート時のリセット対象
| 項目 | リセット内容 |
|---|---|
| マップ | 新規ランダム生成（壁配置含む） |
| プレイヤーHP / 座標 | 初期値・ランダム配置に戻る |
| 敵 | 新規ランダム配置 |
| ターンカウント | 0 |
| スコア | 0 |
| インベントリ（回復アイテム所持数） | 0 |

### 9.2 プレイヤーの状態
| 状態 | 条件 |
|---|---|
| 行動可能 | HP > 0 かつ プレイヤーターン中 |
| 行動不可 | HP > 0 かつ 敵ターン中 |
| 死亡 | HP ≤ 0 |

### 9.3 敵の状態
| 状態 | 条件 |
|---|---|
| 生存 | HP > 0 |
| 撃破済み | HP ≤ 0（マップから除去済み） |

## 11. アイテム仕様（暫定：HP回復薬のみ）

### 11.1 アイテム種類
| ID | 名称 | 効果 | 備考 |
|---|---|---|---|
| ITEM-HP | HP回復薬 | 使用時にHP +`HEAL_AMOUNT` | 暫定値 `HEAL_AMOUNT = 30`（実装後調整） |

> ※ 将来的にアイテム種類が追加される可能性がある（暫定はHP回復薬のみ）。

### 11.2 マップ上の配置
| 項目 | 仕様 |
|---|---|
| 初期配置数 | `INITIAL_ITEM_COUNT`（エリアサイズに依存。§4.6 参照：small=3 / medium=5 / large=8） |
| 配置方式 | ランダム（壁マス・プレイヤー初期マス・敵初期マスを除く） |
| 配置タイミング | ゲーム開始時（マップ生成と同時） |
| ゲーム途中での再配置 | なし |
| フォールバック | 余剰フロアマス数 < `INITIAL_ITEM_COUNT` の場合は配置可能数に切り詰め（エラーにならない） |

### 11.3 取得（拾う）
| 項目 | 仕様 |
|---|---|
| トリガ | プレイヤーが回復薬のあるマスへ移動成立し、かつ インベントリ < MAX_INVENTORY の場合 |
| 効果 | プレイヤーのインベントリ +1、当該マスから回復薬を除去 |
| HPへの影響 | なし（取得しただけでは回復しない） |
| インベントリ上限 | `MAX_INVENTORY = 2` |
| 上限到達時の挙動 | **自動取得は発生しない**。マスの回復薬は **残存** する。プレイヤーが `H`（使用）または `X`（破棄）でインベントリに空きを作った後、再度同じマスへ移動成立すれば取得可能 |

### 11.4 使用（H キー）
| 項目 | 仕様 |
|---|---|
| 入力キー | `H` |
| 効果 | HP += HEAL_AMOUNT（MAX_HPでクランプ） |
| インベントリ消費 | -1 |
| ターン消費 | あり（移動と同様、敵のターンが進む） |

### 11.5 破棄（X キー）
インベントリ満タン時に新しいアイテムを取得するための空き枠作成手段として導入。

| 項目 | 仕様 |
|---|---|
| 入力キー | `X` |
| 効果 | インベントリ -1（HPは変動しない） |
| 破棄対象 | 暫定でアイテム種類は1種のみのため、所持している任意の1個 |
| ターン消費 | あり（敵のターンが進む） |

### 11.6 使用・破棄の正常系
- 使用：インベントリ ≥ 1 かつ HP > 0 → 使用成立、HP回復、インベントリ -1、ターン消費
- 破棄：インベントリ ≥ 1 → 破棄成立、インベントリ -1、ターン消費

### 11.7 使用・破棄の異常系
| ケース | 処理 |
|---|---|
| インベントリ = 0 で `H` 入力 | 使用不成立。ターン消費なし、HP変動なし |
| インベントリ = 0 で `X` 入力 | 破棄不成立。ターン消費なし |
| HP = MAX_HP で `H` 入力 | **使用成立**。HP変動なし（回復量0）、インベントリ -1、ターン消費 |
| プレイヤー敗北後の `H` / `X` 入力 | 無視 |

### 11.8 アイテムの境界値
| 観点 | 境界 | 期待結果 |
|---|---|---|
| 回復後HPが上限超過 | HP=90, HEAL=30 | HP=100でクランプ（オーバーフロー20は破棄） |
| 回復後HPが上限ピッタリ | HP=70, HEAL=30 | HP=100、消費は通常通り発生 |
| HP満タンで使用 | HP=100, HEAL=30 | HP=100のまま、インベントリ -1、ターン消費（回復量0でも成立） |
| インベントリ0で使用 | インベントリ=0 | 使用不成立 |
| インベントリ1で使用 | インベントリ=1 | 使用成立、インベントリ=0 |
| インベントリ満タンで取得試行 | インベントリ=2、回復薬マスへ移動 | 自動取得発生せず、移動のみ成立、マス上にアイテム残存 |
| 破棄後に同マスで取得 | インベントリ=2 → X で破棄 → 同マスへ再移動 | 取得成立、インベントリ=2 |
| 破棄→使用の連続 | インベントリ=2、X → 同ターン中にH不可、次ターンでH | 各操作1ターンずつ消費 |

## 16. 敵スポーン仕様

### 16.1 目的
戦闘機会を増やし、ミス・クリティカルの確率検証をしやすくする。

### 16.2 スポーンパラメータ
| 定数 | 値 | 備考 |
|---|---|---|
| `ENEMY_SPAWN_INTERVAL` | 5 | スポーンが発生するターン間隔 |
| `MAX_ACTIVE_ENEMIES` | 3 | 同時生存敵数の上限 |

### 16.3 発動条件
以下の **両方** を満たすとき、スポーンが発生する。

1. `turn % ENEMY_SPAWN_INTERVAL == 0`（例: turn=5, 10, 15, ...）
2. 現在の生存敵数 < `MAX_ACTIVE_ENEMIES`

条件を満たさない場合（上限到達・候補マスなし）、スポーンは発生せずゲームは継続する。

### 16.4 スポーン位置の選定
以下をすべて除外したマス集合からランダムに1マスを選ぶ。

| 除外条件 | 理由 |
|---|---|
| 壁マス | 通行不可 |
| プレイヤー現在位置 | 重なり不許可 |
| 生存中の敵の位置 | 重なり不許可（§7.2.4 と同様） |
| 回復薬が存在するマス | アイテム消失防止 |
| プレイヤー現在位置からBFS到達不可能なマス | 到達不可の敵を生成しない |

候補マスが存在しない場合、そのターンのスポーンは発生しない。

### 16.5 スポーン後の敵ステータス
| 項目 | 値 |
|---|---|
| HP | `ENEMY_MAX_HP`（= 30） |
| ATK | `ENEMY_ATK`（= 10） |
| 行動開始タイミング | スポーンの**次ターン** [4]から（スポーンしたターンは静止） |

### 16.6 表示
スポーンが発生したターン、画面下部のメッセージ欄に `敵が出現した！` を表示する。

### 16.7 他機能との相互作用
| 項目 | 仕様 |
|---|---|
| 戦闘ロジック | 既存の §8 と完全に同一 |
| 勝敗判定 | スポーン（[6b]）は勝敗判定 [7] より前に実行されるが、スポーンが勝敗条件を変化させることはない |
| リスタート | `R` キーでリスタートするとスポーン済み敵も含めて全てリセットされる |

## 17. 敵ゆるい追尾 AI 仕様

### 17.1 目的
戦闘機会を増やし、ミス・クリティカル・HP 境界値の検証をしやすくする。完全ランダム移動に比べてゲームテンポを向上させる。

### 17.2 パラメータ
| 定数 | 値 | 備考 |
|---|---|---|
| `ENEMY_CHASE_RATE` | 0.7 | 追尾モードに入る確率（0.0〜1.0）。`config.py` の `_DEFAULTS` に定義 |

### 17.3 追尾アルゴリズム（§7.2.2 の段階 1）
各敵が行動するとき、`random.random() < ENEMY_CHASE_RATE` を評価する。

- **True（追尾モード）**:
  1. 上下左右の4方向について「移動後のマンハッタン距離 < 現在のマンハッタン距離」となる方向を列挙する（追尾候補）。
  2. 追尾候補から無効マス（壁・範囲外・他の敵）を除外する。プレイヤーのマスは除外しない。
  3. 有効な追尾候補があれば、その中からランダムに1方向を選んで移動（またはプレイヤーマスなら戦闘）。
  4. 有効な追尾候補がない場合、ランダム移動にフォールバック（§7.2.2 段階 2）。
- **False（非追尾モード）**:
  - §7.2.2 段階 2 のランダム移動を実行する。

### 17.4 距離計算
| 項目 | 仕様 |
|---|---|
| 距離の種類 | **マンハッタン距離**（`|ex - px| + |ey - py|`）|
| 評価タイミング | 各敵が行動する直前の座標で計算 |

### 17.5 移動制約（追尾モード・ランダムモード共通）
| 項目 | 仕様 |
|---|---|
| 壁マス | 移動不可 |
| マップ範囲外 | 移動不可 |
| 他の敵がいるマス | 移動不可（§7.2.4） |
| プレイヤーのマス | 有効。移動成立し戦闘発生（§8） |

### 17.6 テスト観点
| ID | 観点 | 期待結果 |
|---|---|---|
| N-29 | 敵が近づくことがある | 複数ターン観察すると敵がプレイヤー方向へ近づくケースがある |
| N-30 | ENEMY_CHASE_RATE=1.0 の追尾保証 | 障害物のない状況で、必ずマンハッタン距離が短くなる方向へ移動する |
| N-31 | ENEMY_CHASE_RATE=0.0 の完全ランダム | 追尾モードが発動せず従来通りランダム移動になる |
| N-32 | 追尾候補なし時のフォールバック | 近づける方向が全て無効マスの場合、ランダム移動にフォールバックし無効マスへは移動しない |
| E-18 | プレイヤー隣接時の戦闘発生 | 追尾モードで敵がプレイヤーマスへ移動すると戦闘が発生する |
| B-40 | CHASE_RATE=1.0 の距離単調減少 | 追尾候補が存在する限り、毎ターン必ずマンハッタン距離が1縮まる |
| B-41 | 壁封鎖時の無効マス回避 | 追尾方向が全て壁で塞がれているとき、壁マスへ移動しない |
